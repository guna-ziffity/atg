IE6 Form Leaks

This page is for the form leak bug located at 144261

IE6 doesn't allow full deletion of form nodes, this results in increasing dom counts, orphans and memory usage as content is loaded and deleted that contains form nodes.

The two test pages contain a contentPane widget that will reload with content every 2 seconds, the two files that it switches between contain that exact same amount of HTML nodes.

The first test page is the control the content loaded contains only DIV nodes

The test page loads content that contains mostly FORM nodes. In the bug the number of DOM nodes, orphans and memory usage would rise rapidly as the content is loaded and reloaded. The bug is fixed if there is no DOM node increase, no orphaned nodes and the memory graph is the same as when loading just DIVS

These pages must be loaded into something that measures the above values, Drip / sIEve is one of the more common tools for this. They can be downloaded from Sourceforge or the original sIEve site.
